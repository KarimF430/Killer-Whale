Architecture Pattern

  • Frontend: Next.js 15 with App Router (SSR/ISR/CSR hybrid)
  • Backend: Express.js REST API (Separate Node server)
  • Database: MongoDB with Mongoose ODM
  • Caching: Redis (Distributed caching with SWR)
  • Storage: Cloudflare R2 (S3-compatible object storage)
  • Monitoring: Sentry (Error tracking & Performance monitoring)

  • Backend: Render / Self-hosted (Port 5001)
  • Database: MongoDB Atlas / Self-hosted

  • CDN: Cloudflare R2 for images

üöÄ CACHING STRATEGY (Deep Dive)

Three-Layer Caching:

Layer 1: Redis (Backend)

  • Location: Backend middleware
  • Scope: All API responses
  • TTL: 15 min to 2 hours (endpoint-specific)
  • Features:
- Stale-While-Revalidate (SWR) - Cache stampede prevention (Redis locks) - Hierarchical keys with versioning - Automatic warmup on server start

Layer 2: ISR (Next.js)

  • Location: Frontend (Next.js)
  • Scope: Static pages (Home, Brand, Model)
  • TTL: 30 min to 1 hour
  • Features:
- On-demand revalidation - Stale cache served while regenerating - CDN edge caching (Vercel)

Layer 3: Browser Cache

  • Location: User's browser
  • Scope: Images, static assets
  • TTL: Set by Cache-Control headers
  • Features:
- Image lazy loading - Service worker (planned)

Cache Invalidation:
